변수 (Variable)

        * 메모리에 값을 계속 바꿔가면서 저장할 수 있는 공간

        * 변수에 이름을 붙이고 원하는 값을 대입하면 
          필요한 곳에서 변수명을 호출하여 저장된 값을 사용할 수 있다.

        
변수명 명명 규칙

        - 의미 있는 이름을 작성 (대충 a, b 이렇게 짓는 것을 지양)
        - 대소문자 구분
        - 특수문자는 _ 만 사용 가능 (첫 글자 제외 ex : test_case OK / _test NO)
        - 숫자 가능 (첫 글자 제외)
        - 카멜 표기법으로 작성하기
          [카멜 표기법(Camel Case)]
          문장, 여러 단어를 변수명으로 작성할 때 
          이어지는 첫 번째 단어를 대문자로 작성
          ex) user name => userName

JavaScript 변수의 종류와 특징

        var : 변수, 변수명 중복 O (덮어쓰기 가능), 함수 레벨 scope (만 허용)
        let : 변수, 변수명 중복 X 블록{} 레벨 scope
        const : 상수, 변수명 중복 X, 블록{} 레벨 scope

        
사용 우선순위

        1순위 : const (JS는 특정 요소를 선택해서 사용하는 경우가 많기 때문에
                       const 상수에 고정시켜둠)
        2순위 : let 
        3순위 : var (실무에서도 사용 안하는 추세)

var, let, const 의 차이점

        1. var : 변수, 변수 선언 시 중복되는 변수명으로 선언 가능
                 -> 먼저 선언된 변수가 나중에 선언된 변수에 덮어 씌워짐
        2. let : 재선언 불가능, 재할당 가능
                 변수, var의 변수명 중복 문제 해결
        3. const : 재선언, 재할당 모두 불가능
                   오류 문구 : Uncaught TypeError: Assignment to constant variable.
                   상수에 새로운 값을 대입할 수 없는데, 새로운 값이 대입되었다.

함수 레벨 스코프

        함수 내에서 선언된 변수는 함수 내에서만 유효하며, 함수 외부에서는 참조할 수 없다
        즉, 함수 내부에서 선언한 변수는 지역 변수이며, 함수 외부에서 선언한 변수는 전역 변수이다

블록 레벨 스코프

        {} -> 코드 블록, 모든 코드 블록 내에서 선언된 변수는 코드 블록 내에서만 유효하며
        코드 블록 외부에서는 참조할 수 없다. 즉, 코드 블록 내부에서 선언한 변수는 지역 변수이다.

JavaScript의 자료형

        JavaScript는 변수 선언 시 별도의 자료형을 지정하지 않음
        -> 변수에 대입되는 값(리터럴)에 의해서 자료형이 결정됨

        - string (문자열)
        - number (숫자) : 정수 / 실수 모두 포함
        - boolean (논리값) : true / false
        - object (객체) : 배열(Array), 자바스크립트 객체 {key : value, key : value...}
        - function (함수)
        - undefined (정의되지 않은 변수) -> 변수가 선언만 됐고, 값이 대입되지 않음

        (참고)
        - null : 타입보단 값(리터럴)의 개념, 참조하는게 없음
                 ex) document.getElementById("id=오타일경우") -> 가장 흔한 경우
                     id가 일치하는 요소가 없어서 참조할 수 없다

JS 자료형 확인하기

        typeof : 변수 / 값의 자료형을 출력하는 연산자

        사용 방법 : console.log(typeof 해당 함수명)

        ex)
        undefined : 정의되지 않은 변수 / 값이 아직 대입되지 않았다
                    let undef; // 변수 선언
                    console.log("undef : ", undef);
                    console.log("undef : ", undef, typeof undef); // Javascript 식
                    console.log(typeof undef); // Javascript 식
                    console.log(typeof(undef)); // Java식
